## Confirmed Measurement Findings

1. `scripts/v85_grand_sweep_6y.py:evaluate` reconstructs equity using `eq * 0.01 * return_pct`. This is synthetic 1% sizing and does not use emitted trade `pnl`, despite the sweep runner setting `--base-position-pct 0.50`.
2. `v83_balanced` and `v84_config2` have identical strategy-specific arguments and identical shared `BASE` arguments. They are config aliases.
3. V8.5 `BASE` forces lookahead-safe `--entry-lag-bars 1`, while historical V8.4 reported results include legacy runner assumptions that must be reproduced explicitly.
4. Runner costs default to 5 bps fee and 1 bps slippage per side. The old grand-sweep registry did not expose those assumptions in its result file.
5. Profile mode extends `BAR_EXIT` from the normal horizon to three times the configured horizon. It therefore changes more than the exit trigger taxonomy.
6. Profile exits are checked before standard TPSL exits. This can change which exit reason and fill wins on the same bar.

## Execution Assumptions

| Assumption | Current cached runner behavior |
|---|---|
| Entry lag 0 | Signal-bar close plus adverse entry slippage |
| Entry lag 1 | Next volume-bar open plus adverse entry slippage |
| Exit slippage | Adverse slippage applied to selected exit price |
| Fees | Round-trip fee charged as `notional * fee_bps_per_side * 2 / 10000` |
| Stop fill | Stop level selected before adverse exit slippage |
| Position sizing | Current equity times permission size times risk/profile scalar |
| Compounding | Enabled through current-equity sizing |

## Required Fresh-Run Fields

Historical V8.5 JSONL ledgers contain basic trade PnL, return, exit reason, and MAE/MFE. They do not contain enough bar-path state to prove whether a profile exit later would have hit the old target. Fresh V8.6 runs must emit or reconstruct those counterfactual fields before answering the POC-clipping question conclusively.
